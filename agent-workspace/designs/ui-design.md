# 用户界面设计文档

更新时间：2026-06-26

## 1. 文档目的与范围

本文档基于 `requirements-analysis.md`、`interaction-flow.md`、`core-interaction-flow.md` 与患者端高保真原型，定义患者端两个核心页面的**界面内容与组件排布**：

- **主页（Home / 就诊入口 + 历史中心）**：新建问诊入口、历史就诊记录、复诊再入。
- **工作台页（Workbench / AI 诊疗流程界面）**：聊天时间线 + 流程卡片混排 + 阻塞决策 + 全局打断。

本文档只做：

- 页面信息架构、区块划分、组件排布。
- 各类卡片、消息、状态区的内容字段与可执行动作。
- 桌面 / 移动两套响应式布局。
- 界面状态与 XState 流程状态的映射。

本文档**不做**：

- 具体视觉风格（颜色、字号、阴影、圆角等）。这些由 HeroUI 3 + Magic UI + Tailwind 设计变量承载，本文只标注“用哪类组件”。
- 后端接口协议细节（见 `designs/api.md`）。
- 完整文案与法务合规文本。

## 2. 设计原则与约束

源自需求文档的硬约束，落到界面层：

1. **流程型应用，不是聊天工具**：界面随状态机切换，输入框可用性由当前状态决定。
2. **阻塞卡不可被自动越过**：阻塞态下输入框锁定，必须显式做出选择或“暂不决定”才解锁。
3. **已处理卡片保留在时间线**：所有卡片处理后留痕，显示处理结果，不从历史移除。
4. **全局机制优先级最高**：急症 / 超时 / 主动退出以打断式弹层呈现，盖过主流程。
5. **终止态必须显示终止原因**：急症、转大医院、退出结算三类终止都带原因说明。
6. **完成后继续输入触发复诊**：完成态输入框保持可用，再次输入进入复诊评估而非闲聊。
7. **处置由 AI 决策，患者不选分支**：处置方向不暴露为患者可选项；原型中“侧栏模拟决策”仅作开发调试入口。

## 3. 全局信息架构与路由

```text
主页 /                     就诊入口 + 历史中心
  └─ 新建问诊（输入症状）  ──►  工作台 /visit/:sessionId
  └─ 点击历史记录          ──►  工作台 /visit/:sessionId（只读回看 / 复诊）

工作台 /visit/:sessionId    AI 诊疗流程主界面
  └─ 退出 / 完成           ──►  返回主页（历史记录刷新）
```

- 路由用 React Router 8：`/`（主页）、`/visit/:sessionId`（工作台）。
- 主页输入症状后，前端先创建 session（`chatApi.createSession`），再跳转工作台并把首条症状作为第一条用户消息。
- 工作台是单会话视图，sessionId 决定加载哪次就诊。
- 全局打断弹层不切路由，仅在工作台内以 overlay 呈现。

## 4. 组件来源约定（不含具体风格）

界面只标注“使用哪类组件”，具体样式交给组件库：

| 用途 | 组件来源 |
| --- | --- |
| 按钮、输入框、弹窗、抽屉、列表、标签、头像、进度条、骨架屏 | HeroUI 3 |
| AI 文本逐字浮现、生成中提示、文本入场动画 | Magic UI（typing-animation / text-animate / animated-shiny-text）|
| 图标 | lucide-react |
| 布局栅格、间距、响应式断点 | Tailwind CSS 4 |

约定：本文出现的“卡片 / 按钮 / 标签 / 抽屉”等均指上述组件类型，不在此定义其视觉细节。

## 5. 主页设计（就诊入口 + 历史中心）

### 5.1 角色定位

类 ChatGPT 首页：上方是“开始新问诊”的主入口，下方是历史就诊记录中心。患者从这里发起新问诊、回看既往就诊、对已完成就诊发起复诊。不是营销落地页，无需 banner 式宣传。

### 5.2 桌面布局（≥ 1024px）

```text
┌────────────────────────────────────────────────────────────┐
│  顶栏：Logo「东软云脑智能医疗」          患者头像/昵称 ▾      │
├────────────────────────────────────────────────────────────┤
│                                                              │
│            [ 急症提示条：如有危急症状请直接拨打 120 ]          │  ← 常驻安全提示，非弹层
│                                                              │
│                  你好，我可以帮你做什么？                     │  ← 主标题
│        ┌──────────────────────────────────────────┐          │
│        │  描述你的不适，开始 AI 问诊…        [开始] │          │  ← 主入口大输入框
│        └──────────────────────────────────────────┘          │
│        常见：[ 发烧 ] [ 咳嗽 ] [ 嗓子痛 ] [ 腹痛 ]            │  ← 快捷症状 chips（可选起话）
│                                                              │
│  ── 历史就诊 ───────────────────────────────  [全部 ▾]       │
│  ┌────────────────────┐  ┌────────────────────┐             │
│  │ 急性咽炎            │  │ 进行中…             │             │  ← 历史就诊卡（网格）
│  │ 已完成 · 06-24      │  │ 等待缴费 · 06-26    │             │
│  │ 用药 · 随访已设置   │  │ 检验缴费阻塞中      │             │
│  │ [回看] [复诊]       │  │ [继续就诊]          │             │
│  └────────────────────┘  └────────────────────┘             │
└────────────────────────────────────────────────────────────┘
```

- 主入口大输入框居中、视觉权重最高，回车或点「开始」即创建 session 并跳转工作台。
- 历史区为响应式网格（桌面 2~3 列）。

### 5.3 移动布局（< 768px）

- 单列纵向流：顶栏 → 安全提示条 → 主标题 + 输入框 → 快捷 chips → 历史列表（单列卡片）。
- 历史卡片改为整行列表项，主操作按钮放右侧或卡片底部。
- 主入口输入框宽度铺满，发送按钮内置于输入框右侧。

### 5.4 区块与组件清单

| 区块 | 内容 | 组件 | 备注 |
| --- | --- | --- | --- |
| 顶栏 | Logo、产品名、患者头像/昵称下拉 | HeroUI Navbar + Avatar + Dropdown | 下拉含「我的信息」「退出登录」 |
| 安全提示条 | 急症自救引导（拨打 120 / 前往急诊） | HeroUI Alert（info/warning） | 常驻，非打断弹层，全程可见 |
| 主入口 | 主标题 + 大输入框 + 开始按钮 | HeroUI Input/Textarea + Button | 提交即 `createSession` → 跳转工作台 |
| 快捷症状 | 常见症状 chips | HeroUI Chip（可点击） | 点击把该症状填入输入框，不直接发起 |
| 历史就诊区 | 标题 + 筛选 + 历史卡列表 | HeroUI Card 列表 / 网格 | 见 5.5 |
| 空状态 | 无历史时的占位 | HeroUI 空态插画 + 文案 | 见 5.6 |

### 5.5 历史就诊卡（HistoryCard）内容字段

对应数据对象 `VisitSession`（见需求 §数据对象）。

| 字段 | 来源 | 展示 |
| --- | --- | --- |
| 诊断结论 / 主诉摘要 | Diagnosis.诊断结论 或首条症状 | 卡片标题。未确诊时显示主诉摘要 |
| 就诊状态标签 | VisitSession.当前流程状态 | 状态 Chip，颜色按状态语义 |
| 时间 | 开始时间 / 结束时间 | 副标题 |
| 处置摘要 | TreatmentPlan.类型 | 「用药 / 治疗 / 仅医嘱 / 转诊」一行 |
| 终止原因 | VisitSession.终止原因 | 仅终止态显示（急症 / 超时转诊 / 主动退出）|
| 主操作 | 按状态决定 | 见下方动作映射 |

历史卡按 session 状态决定主操作按钮：

| session 状态 | 状态标签文案 | 主操作 |
| --- | --- | --- |
| `completed` | 已完成 | 「回看」+「复诊」 |
| 进行中（chatting/labDecision/labPayment/diagnosis/treatment*） | 进行中 · 等待缴费 等 | 「继续就诊」 |
| `transferred` | 已转诊 | 「回看」 |
| `emergencyTerminated` | 急症终止 | 「回看」 |
| `exited` | 已退出 | 「回看」 |

- 「继续就诊」：跳转工作台，状态机恢复到该 session 当前状态。
- 「复诊」：跳转工作台，对已完成 session 发起复诊（触发自动复诊评估，回到“是否检验”相关环节）。
- 「回看」：进入工作台只读模式，时间线可滚动，输入框与卡片动作禁用（终止态会话）。

### 5.6 空状态

- 无历史就诊：历史区显示「还没有就诊记录，描述不适即可开始第一次问诊」+ 指向上方输入框的引导。
- 加载中：历史卡用骨架屏（HeroUI Skeleton）。

## 6. 工作台页设计（AI 诊疗流程）

### 6.1 角色定位

患者端核心工作界面。以聊天时间线为主轴，流程卡片混排其中，底部输入框可用性随状态切换，右侧（桌面）/抽屉（移动）展示本次就诊摘要与进度，全局打断以 overlay 呈现。源自原型，扩展为响应式三栏。

### 6.2 桌面布局（≥ 1024px，三栏）

```text
┌──────────────────────────────────────────────────────────────────────┐
│ 顶栏：[医] AI 导诊医生  在线·急症守护并行    ⏱ 24:30   [当前状态] [X 退出]│
├───────────────────────────────────────────────────┬──────────────────┤
│  聊天时间线（消息 + 卡片混排，可滚动）              │  就诊摘要侧栏      │
│                                                     │  ───────────────  │
│  [AI] 你好，请描述你的不适。                        │  本次就诊          │
│              嗓子痛、有点发烧… [患者]               │  状态：等待缴费    │
│  [AI] 建议做一次血常规检验。                        │  ⏱ 总计时 24:30   │
│  ┌─────────────────────────────────────────────┐   │  ───────────────  │
│  │ 阻塞卡：是否检验？  [同意检验][不查]          │   │  进度             │
│  │ 暂不决定 · 继续聊聊                            │   │  ✓ 问诊            │
│  └─────────────────────────────────────────────┘   │  ● 检验缴费        │
│                                                     │  ○ 确诊           │
│                                                     │  ○ 处置           │
│                                                     │  ───────────────  │
│                                                     │  费用             │
│                                                     │  检验费 待支付 ¥35 │
├───────────────────────────────────────────────────┴──────────────────┤
│ [锁定提示条]（阻塞态显示）请先在上方卡片中做出选择                       │
│ [ 请先在上方卡片中选择…                                    ] [发送 ➤]   │
└──────────────────────────────────────────────────────────────────────┘
```

- **左/中主区**：聊天时间线，占主要宽度。
- **右栏**：就诊摘要（状态、计时、阶段进度、费用/检验/处置进度）。原型中此栏被“模拟决策”占用，正式版改为只读摘要；模拟决策仅在开发模式以折叠面板出现。
- **底部**：锁定提示条（按需显示）+ 输入框 + 发送。
- 顶栏的“急症守护并行”是常驻状态文案，提示后台监听持续运行。

### 6.3 移动布局（< 768px，单列）

- 还原原型单列：顶栏 → 聊天时间线（全宽）→ 锁定条 → 输入框。
- 就诊摘要侧栏改为**顶栏可展开的抽屉 / 底部抽屉**（HeroUI Drawer），通过顶栏「状态」chip 或摘要图标唤起。
- 卡片在时间线内 `align-self: stretch` 全宽展开。
- 全局打断弹层（急症 / 超时）全屏遮罩；退出确认为底部 sheet（与原型一致）。

### 6.4 顶栏（TopBar）

| 元素 | 内容 | 组件 | 行为 |
| --- | --- | --- | --- |
| AI 身份 | 头像「医」+「AI 导诊医生」+「在线 · 急症守护并行」 | Avatar + 文本 | 静态 |
| 当前状态 chip | 当前流程状态文案（问诊中 / 等待缴费 / 已确诊…） | HeroUI Chip | 移动端点击可唤起摘要抽屉 |
| 总计时 | 整次导诊倒计时 ⏱ | 文本 + 图标 | 会话级单一总计时，超时触发升级 |
| 退出入口 X | 主动退出 | HeroUI Button（icon） | 始终可用，点击进入退出二次确认 |

- 退出入口在任意状态都必须可点（需求硬约束）。

### 6.5 聊天时间线（ChatTimeline）

消息与卡片同处一条时间线，按创建时间排列，对应数据对象 `Message` + `FlowCard`。

**消息气泡（Message）**

| 类型 | 对齐 | 内容 | 备注 |
| --- | --- | --- | --- |
| AI 消息 | 左 | AI 文本 | 流式生成时用 Magic UI typing-animation；生成中显示「正在输入」 |
| 患者消息 | 右 | 患者输入文本 | — |
| 系统消息 | 居中/弱化 | 状态变更提示（如「已支付 ¥35」「检测到就诊结束后继续对话」） | 轻量样式 |

**生成中状态**：AI 异步生成时显示「正在分析…/正在生成诊断…」占位（animated-shiny-text），可被急症/退出中断并标记失效。

### 6.6 卡片体系（FlowCard）

所有卡片插入时间线，对应 `FlowCard`，含状态：待处理 / 已同意 / 已拒绝 / 已否决 / 已完成 / 已失效。处理后卡片**保留**并显示结果（如「已支付」「已不查」「已失效」角标）。

四大卡片类别：

#### A. 普通信息卡（非阻塞，不锁输入框）

| 卡片 | 内容字段 | 组件 | 对应状态 |
| --- | --- | --- | --- |
| 确诊卡 | 标签「已确诊·置信度」、诊断结论、诊断依据（区分问诊依据 / 检验依据）、风险提示 | Card + Chip + text-animate | `diagnosis` |
| 医嘱卡 | 医嘱内容、复诊建议、观察项、风险提示 | Card | 处置-仅医嘱 |
| 处置安排卡 | 治疗预约 / 用药提醒 / 随访提醒等只读条目 | Card + 图标 | `treatmentExecution` |
| 就诊完成卡 | 「本次就诊已完成」、关键结论摘要、后续注意事项入口 | Card | `completed` |

确诊卡关键规则：**未做检验时，依据只标“来自问诊”，不得伪造检验结论。**

#### B. 阻塞决策卡（阻塞，锁定输入框）

| 卡片 | 内容字段 | 动作 | 对应状态 |
| --- | --- | --- | --- |
| 是否检验卡 | 标签「额外决策·对话已锁定」、建议检验原因、检验项目、费用/待缴信息 | 主：同意检验；次：不查直接看诊断；底部：暂不决定·继续聊聊 | `labDecision` |
| 检验缴费卡 | 标签「缴费·待支付」、项目、金额、医保统筹、实付、支付状态 | 主：确认支付；底部：暂不缴费·继续聊聊；支付中/成功/失败重试 | `labPayment` |
| 药品缴费卡 | 同缴费卡结构，项目为药品清单 | 主：确认支付；底部：暂不缴费 | 处置-用药 |
| 取药方式卡 | 选择到院自取 / 配送到家 | 主/次按钮 + 暂不决定 | 处置-用药 |

阻塞卡通用交互：

- 出现即锁定输入框，显示锁定提示条「请先在上方卡片中做出选择」。
- 「暂不决定 / 暂不缴费」= 否决/挂起 → 解锁输入框，提示可继续对话，后续 AI 可再次出同类卡（pendingRedo 机制）。
- 「不查」≠ 否决，是明确跳过检验直接进入确诊。
- 支付状态：待支付 → 支付中（loading）→ 成功（回填检验/进入下一步）/ 失败（重试）。

#### C. 处置方案卡（AI 决策结果，非阻塞展示）

| 分支 | 内容字段 | 后续 |
| --- | --- | --- |
| 用药 | 标签「处置方案·AI 已决策」、药品名、剂量、用法、禁忌/提醒 | 衔接药品缴费卡 → 取药方式卡 → 用药提醒 |
| 自动化治疗 | 治疗项目、排队/预约时间、注意事项、执行/完成状态 | 衔接处置安排卡 → 完成卡 |
| 仅医嘱 | 医嘱内容、观察项、复诊建议 | 衔接医嘱卡 → 完成卡 |
| 转大医院 | 升级标签、转诊原因、建议去向/下一步指引 | 进入终止态（见 D），不再继续本院处置 |

“正在生成处置方案”阶段显示生成中提示（对应 `treatmentDecision`）。

#### D. 终止卡 / 终止层

| 终止 | 呈现形式 | 内容 | 输入框 |
| --- | --- | --- | --- |
| 急症终止 | 全屏遮罩 overlay（最高优先级） | 「情况紧急」+ 危急信号说明 + 「请立即前往急诊」+「我已知晓」 | 禁用，标记本次线上导诊失效 |
| 超时升级 | 全屏遮罩 overlay | 「本次导诊已超时」+ 建议前往大医院 + 「知道了」 | 关闭后进入转大医院终止卡，禁止继续本院流程 |
| 转大医院 | 时间线终止卡 + 锁定 | 转诊原因、建议去向 | 锁定为终止态 |
| 主动退出 | 底部 sheet 二次确认 → 退出结算 | 见 6.8 | 确认后进入终止/挂起 |

### 6.7 输入区与锁定态（InputBar）

| 状态 | 输入框 | 锁定条 | 触发场景 |
| --- | --- | --- | --- |
| 可用 | 可输入、可发送 | 隐藏 | `chatting`、否决后继续对话、`completed` |
| 阻塞锁定 | 禁用，占位文案=锁定原因 | 显示「请先在上方卡片中做出选择」 | 阻塞决策卡出现时 |
| 决策等待锁定 | 禁用 | 显示「AI 正在决策处置方案」 | `treatmentDecision` |
| 终止锁定 | 禁用，占位「本次就诊已结束」 | 显示完成/终止文案 | 急症/转诊/退出终止 |
| 完成可用 | 可用，占位「仍有不适？继续描述将自动开始复诊…」 | 隐藏 | `completed` |

- 完成态输入框可用，再次输入触发 `FOLLOW_UP_MESSAGE_SENT` → 自动复诊评估（回到是否检验环节），不追加到已完结流程。

### 6.8 主动退出（二次确认 + 退出结算）

源自原型 sheet，分两屏：

1. **确认屏**：标题「确认结束本次就诊？」+ 说明「退出后果由已发生的不可逆动作决定」+ 后果清单：
   - 未缴费/未执行 → 直接结束，无需结算
   - 已缴费未执行 → 原路退款
   - 已执行检验/治疗 → 费用照收，结果留档
   - 已取药/已配送 → 按退药政策处理
   - 动作：「取消，继续就诊」/「确认退出」
2. **完成屏**：「就诊已结束 / 挂起」+ 已按当前阶段结算并保存就诊快照 + 「完成」（返回主页）。

- 退出不可逆；退出期间需处理未支付订单、已支付未完成项目、已生成处置等状态。

### 6.9 就诊摘要侧栏 / 抽屉（VisitSummary）

| 区块 | 内容 |
| --- | --- |
| 当前状态 | 流程状态文案 + 状态 chip |
| 总计时 | 整次导诊倒计时 / 已用时 |
| 阶段进度 | 问诊 → 检验缴费 → 确诊 → 处置 → 完成 的进度指示（✓ 完成 / ● 当前 / ○ 未到）|
| 费用 | 检验费、药品费等待缴/已缴 |
| 检验 | 检验项目与结果状态 |
| 处置 | 处置类型与执行状态 |

- 桌面常驻右栏；移动端为抽屉，由顶栏状态 chip 唤起。
- 开发模式下，“模拟大模型决策”面板作为可折叠调试区出现在侧栏底部，正式构建隐藏。

## 8. 界面状态 ↔ XState 状态映射

| XState 状态 | 工作台界面表现 | 输入框 |
| --- | --- | --- |
| `chatting` | 聊天时间线自由对话 | 可用 |
| `labDecision` | 是否检验阻塞卡 | 锁定 |
| `labPayment` | 检验缴费阻塞卡（支付状态机内含） | 锁定（否决可解锁） |
| `diagnosis` | 确诊卡（信息卡） | 可用（可补充信息） |
| `treatmentDecision` | 「正在生成处置方案」生成中 | 锁定 |
| `treatmentExecution` | 处置方案卡 + 对应执行卡 | 视分支（缴费/取药阻塞，其余可用） |
| `completed` | 就诊完成卡 | 可用（触发复诊） |

| `transferred` | 转大医院终止卡 | 禁用 |
| `exitSettlement` | 退出结算 sheet | 禁用 |
| `exited` | 退出完成屏 → 返回主页 | 禁用 |

关键事件（`MESSAGE_SENT`、`LAB_*`、`DIAGNOSIS_READY`、`TREATMENT_*`、`FOLLOW_UP_MESSAGE_SENT`、`EMERGENCY_DETECTED`、`VISIT_TIMEOUT`、`TRANSFER_REQUIRED`、`EXIT_*`）由界面动作触发，界面只读状态机当前态渲染对应区块。

## 9. 响应式断点策略

| 断点 | 主页 | 工作台 |
| --- | --- | --- |
| < 768px（移动） | 单列：入口 + 历史列表（单列） | 单列：顶栏 + 时间线 + 输入框；摘要为抽屉；终止全屏遮罩 |
| 768–1024px（平板） | 历史 2 列网格 | 时间线全宽，摘要折叠为可展开侧抽屉 |
| ≥ 1024px（桌面） | 居中入口 + 历史 2~3 列网格 | 三栏：时间线 + 常驻摘要右栏 |

- 移动端尽量保持原型的单列体验与底部 sheet/全屏遮罩交互。
- 工作台主区在大屏限制最大宽度，避免聊天行过长。

## 10. 可访问性要点

- 所有按钮、输入框使用 HeroUI 可访问性组件，保留 `aria-label`（退出、发送等图标按钮）。
- 阻塞锁定态对屏幕阅读器播报锁定原因（`aria-live` 提示条）。
- 急症 / 超时 overlay 出现时设置焦点陷阱（focus trap），并以 `role="alertdialog"` 高优先级播报。
- 状态 chip 不只用颜色区分语义，附带文字标签。
- 流式生成文本提供「生成中 / 已完成」可感知状态，避免仅靠动画判断。

## 11. 待确认问题（影响界面细节）

- 检验缴费卡“否决”的业务后果文案（取消检验 / 暂缓 / 重新建议）—— 影响否决后系统消息与摘要状态。
- 确诊后患者补充信息是否允许重新诊断 —— 影响确诊卡是否提供「重新评估」动作。
- 自动化治疗分支真实执行/排队/完成字段 —— 影响治疗卡进度展示粒度。
- 用药分支是否含处方/药师审核/配送 —— 影响用药卡与缴费卡步骤数。
- 急症命中文案与法律责任边界 —— 影响急症 overlay 文案。
- 总计时时长、起点、暂停/重入规则 —— 影响顶栏计时与超时触发展示。
- 主动退出“结束 vs 挂起”判定 —— 影响退出完成屏文案与历史卡状态标签。
