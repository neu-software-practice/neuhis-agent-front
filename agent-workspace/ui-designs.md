# UI 设计文档

更新时间：2026-06-26

产品名：**东软云脑智能医疗**

## 1. 总览

本设计文档基于 `requirements-analysis.md`、`interaction-flow.md`、`core-interaction-flow.md` 和 `无人医院_患者端_交互原型.html`。文档目标不是复述状态机，而是从患者使用任务出发，定义首页和问诊 Agent 工作台的界面结构、信息优先级、ASCII Art 草图、关键交互和跳转关系。

本文件不定义具体组件 props、色值、字号、动效参数和响应式断点。HeroUI 3、Magic UI、lucide-react 只在适合落地的地方标注使用边界。

### 1.1 设计判断

患者进入系统时的核心问题不是“系统有哪些状态”，而是：

- 我现在该从哪里开始？
- 我能不能少打字，把症状说清楚？
- AI 为什么问我这些，下一步是否必须做？
- 现在流程卡住在哪里，我不想做会怎样？
- 如果情况紧急、超时、退出，后果是什么？
- 我以前看过什么病，应该在哪里找完整记录？

因此界面遵循以下原则：

- **主线优先**：工作台只保留当前问诊所需信息，不在聊天区堆完整历史。
- **用户触发**：历史、病史、检验说明、费用明细等次级信息默认收起，由患者主动展开。
- **输入减负**：刚进入工作台和 AI 追问时，提供症状、时长、程度、是否有危险信号等快捷 chip；chip 用于填充或补全输入，不替代患者最终确认。
- **渐进露出**：常见症状 chip、快速回答、病史摘要只在空输入、初始问诊或相关追问阶段出现；患者开始输入、滚动回看或出现阻塞卡后自动弱化或隐藏。
- **避免喧宾夺主**：任何辅助控件都不能压过时间线和当前必须处理的卡片。阻塞卡出现时，只突出卡片动作和锁定原因。
- **安全清晰**：急症、超时、退出和转大医院必须明确后果，且优先级高于普通对话体验。

### 1.2 页面清单

| ID | 名称 | 类型 | 说明 |
| --- | --- | --- | --- |
| `HOME-LANDING` | 首页 Landing 态 | 首页 | 默认入口，聚焦新建问诊、继续未完成、最近一次复诊提醒 |
| `HOME-HISTORY` | 历史就诊列表 | 首页 | 完整历史集中在首页历史页，不放入工作台主界面 |
| `HOME-PROFILE` | 个人中心/设置 | 首页 | 患者资料、病史维护、过敏史、医保、通知和隐私 |
| `WB-LOADING` | 上下文加载态 | 工作台 | 新出诊/复诊进入时读取病史、上次诊断和会话资料 |
| `WB-CHATTING` | 正常问诊对话态 | 工作台 | AI 问诊、患者输入、快捷症状 chip、快速回答和分析状态 |
| `WB-CARD-BLOCKED` | 阻塞卡锁定输入态 | 工作台 | 是否检验、缴费、取药方式等必须先处理的卡 |
| `WB-DIAGNOSIS` | 确诊卡展示态 | 工作台 | 展示诊断、依据、风险信号和下一步处置生成 |
| `WB-TREATMENT` | 处置执行卡态 | 工作台 | 用药、治疗、仅医嘱等 AI 决策后的执行卡 |
| `WB-COMPLETED` | 就诊完成态 | 工作台 | 完成总结，输入仍可用，继续输入触发复诊 |
| `WB-EMERGENCY` | 急症打断 Overlay | 全局覆盖 | 急症守护命中后最高优先级终止 |
| `WB-TIMEOUT` | 超时升级 Overlay | 全局覆盖 | 整次导诊总超时后提示前往大医院 |
| `WB-EXIT` | 主动退出 Sheet | 全局覆盖 | 顶栏退出后的确认和结算结果 |
| `WB-TRANSFER` | 转大医院终止卡 | 工作台 | 达上限、本院能力不足或超时后的升级终止 |

### 1.3 路由映射

| 路由 | 默认界面 | 可达界面状态 |
| --- | --- | --- |
| `/` | `HOME-LANDING` | 可切换到 `HOME-HISTORY`、`HOME-PROFILE` |
| `/history` | `HOME-HISTORY` | 可继续就诊、发起复诊、回看记录 |
| `/profile` | `HOME-PROFILE` | 可维护患者资料并返回首页 |
| `/workbench/new` | `WB-LOADING` | 新出诊上下文加载后进入 `WB-CHATTING` |
| `/workbench/:sessionId` | `WB-LOADING` | 继续就诊或复诊上下文加载后恢复工作台状态 |

全局覆盖态不单独作为路由。`WB-EMERGENCY`、`WB-TIMEOUT`、`WB-EXIT` 从工作台任意非终止状态触发。完整历史只在 `/history`；工作台移动端只保留“本次上下文摘要”，默认收起。

### 1.4 工作台信息架构

移动端固定宽度参考 375pt，视觉结构与 340px 手机框原型一致：

```text
┌─────────────────────────────────────┐
│ 顶栏：AI头像 名称 急症守护 倒计时 退出  │
├─────────────────────────────────────┤
│ 本次上下文摘要条：新出诊/复诊 · 已读取  │
│   移动端默认收起，点击展开摘要          │
├─────────────────────────────────────┤
│ 虚拟滚动时间线：消息 + 卡片 + 系统事件  │
│   不展示完整历史，只展示本次会话内容     │
├─────────────────────────────────────┤
│ 输入辅助区：症状chip/快速回答/补充资料  │
│   仅在相关阶段、空输入或用户触发时出现   │
├─────────────────────────────────────┤
│ 锁定栏：仅阻塞卡出现时显示              │
├─────────────────────────────────────┤
│ 底部输入：文本/语音/附件入口 + 发送      │
└─────────────────────────────────────┘
```

PC 端可以在主列右侧放“本次摘要侧栏”，但侧栏默认窄化，不放完整历史列表。完整历史仍由 `HOME-HISTORY` 承载。

组件标注：

- HeroUI：Button、Card、Modal、Drawer、Tabs、Input、Chip、Accordion、Progress、Skeleton、Listbox。
- Magic UI：只用于 AI 正在生成、上下文加载、轻量文字反馈，不用于大面积装饰。
- lucide-react：发送、语音、加号、时钟、退出、警示、锁、展开/收起等图标。
- React Virtuoso：工作台时间线虚拟滚动。

关键交互：

- “继续上次问诊”只在存在进行中会话时出现，并优先放在首页顶部。
- “新建问诊”进入 `/workbench/new`；若患者先点了常见症状 chip，则把症状作为首条草稿带入工作台，不直接替患者发送。
- 常见入口是减少首次输入负担，不是诊断分流。点击“发烧”等 chip 后进入工作台，输入框预填或首屏显示已选症状，可编辑后发送。
- 最近一次就诊只展示一条摘要和复诊入口；完整历史入口在底部 Tab 的“历史”。
- 未登录或身份未核验时，任何进入问诊的动作先完成身份核验，再回到目标流程。

避免喧宾夺主：

- 首页可以展示常见症状入口，因为这是进入任务前的选择；进入工作台后，完整入口列表不再常驻。
- 如果有进行中会话，“继续就诊”优先于新建问诊，避免患者误开新会话。

## 3. 工作台系列

### 3.1 工作台通用框架

```text
┌─────────────────────────────────────┐
│  医  AI导诊医生       急症守护  24:30  X │
│     在线 · 新出诊                    │
├─────────────────────────────────────┤
│ 本次上下文  已读取病史  [展开]          │
├─────────────────────────────────────┤
│                                     │
│  AI/患者消息、流程卡片、系统事件        │
│                                     │
├─────────────────────────────────────┤
│ 快捷辅助：随阶段变化，可自动隐藏         │
├─────────────────────────────────────┤
│ [ + ] [ 输入框................ ] [发送] │
└─────────────────────────────────────┘
```

移动端上下文规则：

- 默认只显示一行“本次上下文摘要”，例如“新出诊 · 已读取病史”或“复诊 · 引用上次急性咽炎记录”。
- 点击“展开”打开小型 Drawer/Accordion，仅展示本次相关摘要：入口类型、过敏史提示、上次诊断摘要、未完成事项。
- 不在工作台展示完整历史列表；完整历史必须回到 `HOME-HISTORY`。
- 如果患者从历史进入复诊，上下文摘要应告诉患者“已引用哪次记录”，避免患者担心 AI 没看旧资料。

输入辅助规则：

- 输入框为空、未出现阻塞卡、患者处于初始描述或 AI 追问时，显示快捷 chip。
- 患者开始输入、发送后、上滑回看、卡片阻塞、Overlay 出现时，快捷 chip 自动收起。
- chip 的作用是填入草稿、追加短语或作为快速回答；除非文案明确为按钮动作，否则不直接提交。
- “+”入口打开补充资料 Sheet，可添加体温、持续时间、疼痛程度、图片/报告等；本期可先设计入口，不强制实现附件上传。

关键交互：

组件标注：HeroUI Progress/Skeleton/Card，Magic UI 可用于短文本加载反馈。

分析中：

```text
┌─────────────────────────────────────┐
│  医  AI导诊医生       急症守护  23:18  X │
│     在线 · 正在分析                  │
├─────────────────────────────────────┤
│                              患者  38.5℃ │
│ ┌─────────────────────────────────┐ │
│ │ AI 正在汇总：病史、回答、危险信号   │ │
│ └─────────────────────────────────┘ │
│ AI  为区分细菌或病毒感染，建议...      │
├─────────────────────────────────────┤
│ [ + ] [ 继续补充症状.......... ] [发送] │
└─────────────────────────────────────┘
```

关键交互：

- 症状 chip 在初始空输入时出现；点击后写入草稿，例如“我发烧”，患者可继续补充“从昨晚开始”再发送。
- “我不知道怎么描述”触发引导式输入，AI 先问部位、时长、程度、伴随症状。
- AI 追问可生成上下文相关快速回答，如“有/没有”“今天开始/超过 3 天”“轻微/中等/严重”。
- “+”入口提供补充资料 Sheet：体温、持续时间、疼痛评分、用药史、图片/报告入口。患者触发后才出现，不常驻展开。
- Agent 分析状态以系统事件插入时间线，避免患者误以为页面卡死。
- 若信息不足且需追问，继续留在本态；若需客观依据，进入 `WB-CARD-BLOCKED`；若追问达上限，进入 `WB-TRANSFER`。
- 每次患者输入、AI 回复、检验结果回填和随访变化都触发急症守护筛查。

避免喧宾夺主：

- chip 区不超过两行；超过数量放入“更多症状”Sheet。
- 时间线滚动回看时隐藏 chip，保留输入框。
- 阻塞卡出现时隐藏全部输入辅助，只显示锁定栏和卡片动作。

## 5. 输入辅助设计

输入辅助不是独立页面，但它直接影响患者能否顺利开始问诊。

| 场景 | 展示内容 | 触发 | 消失条件 |
| --- | --- | --- | --- |
| 首次进入问诊 | 常见症状 chip：发烧、咳嗽、咽痛、头痛、腹痛等 | `WB-CHATTING` 初始空输入 | 患者开始输入、发送、滚动回看、进入阻塞卡 |
| 患者不知道怎么说 | “我不知道怎么描述” chip | 初始空输入 | 点击后进入引导式追问 |
| AI 问是否有某症状 | 有/没有/不确定 chip | AI 追问生成 | 回答后消失 |
| AI 问时长/程度 | 今天开始、1-3 天、超过 3 天、轻微/中等/严重 | AI 追问生成 | 回答后消失 |
| 需要客观补充 | 体温、疼痛评分、用药史、图片/报告入口 | 点击输入框左侧 `+` | Sheet 关闭后消失 |
| 完成后复诊 | 症状加重、出现新症状、药后不适 | `WB-COMPLETED` | 触发复诊后消失 |

规则：

- chip 只帮助组织语言，不直接替患者确认医疗决策。
- 与阻塞卡动作冲突的 chip 必须隐藏。
- chip 文案使用患者语言，不使用内部状态名。
- 快捷输入不能制造事实；例如不能默认填“无呼吸困难”，必须由患者点击确认。

## 7. 覆盖检查

- 首页覆盖新出诊、常见症状入口、继续就诊、最近一次复诊提醒和完整历史入口。
- 完整历史集中在 `HOME-HISTORY`；工作台移动端历史默认收起，仅展示本次上下文摘要。
- 工作台覆盖上下文加载、初始问诊、快捷症状 chip、快速回答、补充资料入口、Agent 分析、阻塞卡、检验缴费、结果回填、确诊、处置、完成。
- 阻塞卡覆盖同意、不查、暂不决定、查看说明、缴费成功/失败/暂缓和处理留痕。
- 全局机制覆盖急症守护、整次导诊单一超时和主动退出结算。
- 终止态覆盖急症终止、超时/转大医院升级、追问或检验上限、本院能力不足。
- 完成后继续输入和复诊 chip 覆盖自动复诊，不把新输入追加到已完成会话。
- ASCII 草图保持原型的固定顶栏、时间线混排、锁定栏、底部输入、Overlay 和 Sheet 结构，同时补充患者输入减负和渐进式信息架构。
