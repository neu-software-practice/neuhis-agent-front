# UI 设计文档

更新时间：2026-06-26

产品名：**东软云脑智能医疗**

## 1. 总览

本设计文档基于 `requirements-analysis.md`、`interaction-flow.md`、`core-interaction-flow.md` 和 `无人医院_患者端_交互原型.html`。文档目标不是复述状态机，而是从患者使用任务出发，定义首页和问诊 Agent 工作台的界面结构、信息优先级、ASCII Art 草图、关键交互和跳转关系。

本文件不定义具体组件 props、色值、字号、动效参数和响应式断点。HeroUI 3、Magic UI、lucide-react 只在适合落地的地方标注使用边界。

### 1.1 设计判断

患者进入系统时的核心问题不是“系统有哪些状态”，而是：

- 我现在该从哪里开始？
- 我能不能少打字，把症状说清楚？
- AI 为什么问我这些，下一步是否必须做？
- 现在流程卡住在哪里，我不想做会怎样？
- 如果情况紧急、超时、退出，后果是什么？
- 我以前看过什么病，应该在哪里找完整记录？

因此界面遵循以下原则：

- **主线优先**：工作台只保留当前问诊所需信息，不在聊天区堆完整历史。
- **用户触发**：历史、病史、检验说明、费用明细等次级信息默认收起，由患者主动展开。
- **输入减负**：刚进入工作台和 AI 追问时，提供症状、时长、程度、是否有危险信号等快捷 chip；chip 用于填充或补全输入，不替代患者最终确认。
- **渐进露出**：常见症状 chip、快速回答、病史摘要只在空输入、初始问诊或相关追问阶段出现；患者开始输入、滚动回看或出现阻塞卡后自动弱化或隐藏。
- **避免喧宾夺主**：任何辅助控件都不能压过时间线和当前必须处理的卡片。阻塞卡出现时，只突出卡片动作和锁定原因。
- **安全清晰**：急症、超时、退出和转大医院必须明确后果，且优先级高于普通对话体验。

### 1.2 页面清单

| ID | 名称 | 类型 | 说明 |
| --- | --- | --- | --- |
| `HOME-LANDING` | 首页 Landing 态 | 首页 | 默认入口，聚焦新建问诊、继续未完成、最近一次复诊提醒 |
| `HOME-HISTORY` | 历史就诊列表 | 首页 | 完整历史集中在首页历史页，不放入工作台主界面 |

| `WB-CARD-BLOCKED` | 阻塞卡锁定输入态 | 工作台 | 是否检验、缴费、取药方式等必须先处理的卡 |

| `WB-EMERGENCY` | 急症打断 Overlay | 全局覆盖 | 急症守护命中后最高优先级终止 |
| `WB-TIMEOUT` | 超时升级 Overlay | 全局覆盖 | 整次导诊总超时后提示前往大医院 |
| `WB-EXIT` | 主动退出 Sheet | 全局覆盖 | 顶栏退出后的确认和结算结果 |
| `WB-TRANSFER` | 转大医院终止卡 | 工作台 | 达上限、本院能力不足或超时后的升级终止 |

### 1.3 路由映射

| 路由 | 默认界面 | 可达界面状态 |
| --- | --- | --- |
| `/` | `HOME-LANDING` | 可切换到 `HOME-HISTORY`、`HOME-PROFILE` |
| `/history` | `HOME-HISTORY` | 可继续就诊、发起复诊、回看记录 |
| `/profile` | `HOME-PROFILE` | 可维护患者资料并返回首页 |
| `/workbench/new` | `WB-LOADING` | 新出诊上下文加载后进入 `WB-CHATTING` |
| `/workbench/:sessionId` | `WB-LOADING` | 继续就诊或复诊上下文加载后恢复工作台状态 |

全局覆盖态不单独作为路由。`WB-EMERGENCY`、`WB-TIMEOUT`、`WB-EXIT` 从工作台任意非终止状态触发。完整历史只在 `/history`；工作台移动端只保留“本次上下文摘要”，默认收起。

### 1.4 工作台信息架构

移动端固定宽度参考 375pt，视觉结构与 340px 手机框原型一致：

```text
┌─────────────────────────────────────┐
│ 顶栏：AI头像 名称 急症守护 倒计时 退出  │
├─────────────────────────────────────┤
│ 本次上下文摘要条：新出诊/复诊 · 已读取  │
│   移动端默认收起，点击展开摘要          │
├─────────────────────────────────────┤
│ 虚拟滚动时间线：消息 + 卡片 + 系统事件  │
│   不展示完整历史，只展示本次会话内容     │
├─────────────────────────────────────┤
│ 输入辅助区：症状chip/快速回答/补充资料  │
│   仅在相关阶段、空输入或用户触发时出现   │
├─────────────────────────────────────┤
│ 锁定栏：仅阻塞卡出现时显示              │
├─────────────────────────────────────┤
│ 底部输入：文本/语音/附件入口 + 发送      │
└─────────────────────────────────────┘
```

PC 端可以在主列右侧放“本次摘要侧栏”，但侧栏默认窄化，不放完整历史列表。完整历史仍由 `HOME-HISTORY` 承载。

组件标注：

- HeroUI：Button、Card、Modal、Drawer、Tabs、Input、Chip、Accordion、Progress、Skeleton、Listbox。
- Magic UI：只用于 AI 正在生成、上下文加载、轻量文字反馈，不用于大面积装饰。
- lucide-react：发送、语音、加号、时钟、退出、警示、锁、展开/收起等图标。
- React Virtuoso：工作台时间线虚拟滚动。

## 2. 首页系列

### 2.1 `HOME-LANDING` 首页 Landing 态

首页遵循”一屏一任务”原则。患者打开 App 的核心诉求只有一个：开始或继续问诊。首页不堆叠历史摘要、复诊入口等次级信息——这些已有专门的 Tab 承载。

```text
┌─────────────────────────────────────┐
│ 东软云脑智能医疗              头像/登录 │
├─────────────────────────────────────┤
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 继续上次问诊                     │ │
│ │ 发热、咽痛 · 进行中              │ │
│ │ [ 继续就诊 ]                     │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 今天哪里不舒服？                      │
│ ┌─────────────────────────────────┐ │
│ │ [ 输入框：描述症状........... ]    │ │
│ │ [ 发烧 ] [ 咳嗽 ] [ 咽痛 ] [更多] │ │
│ │                                 │ │
│ │ [ 开始问诊 ]                     │ │
│ └─────────────────────────────────┘ │
│                                     │
├─────────────────────────────────────┤
│ 首页              历史              我的 │
└─────────────────────────────────────┘
```

关键交互：

- “继续上次问诊”只在存在进行中会话时出现，置于首屏顶部。无进行中会话时该卡片不渲染，首屏只有输入区域。
- 首页核心是一个输入区域：患者可以直接打字描述症状，也可以点击常见症状 chip 填入输入框（草稿 chip 行为），然后点”开始问诊”进入工作台。
- 常见症状 chip 默认只展示一行（3-4 个高频项），点击”更多”展开第二行。不在首屏堆两行 chip。
- “开始问诊”进入 `/workbench/new`，带上输入框内容作为首条草稿。如果输入框为空，也可直接进入工作台空输入态。
- 最近一次就诊、复诊入口不在首页展示——它们在”历史” Tab 中可达，避免首屏信息过载。
- 未登录或身份未核验时，任何进入问诊的动作先完成身份核验，再回到目标流程。

设计意图：

- 首页视觉重心只有一个动作入口，降低患者决策负担。
- 如果有进行中会话，”继续就诊”优先于新建输入区域的视觉层级，避免患者误开新会话。
- 复诊、历史回看等次级功能不在首页抢注意力，由底部 Tab 承载。

```text

├─────────────────────────────────────┤

│ │ 当前：等待是否检验确认 · 24:30      │ │
│ │ [ 继续就诊 ]                      │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 昨天                                  │
│ ┌─────────────────────────────────┐ │
│ │ 已完成 · 急性咽炎                 │ │
│ │ 处置：用药 · 随访提醒已开启         │ │
│ │ [ 发起复诊 ] [ 回看记录 ]          │ │

│ ┌─────────────────────────────────┐ │

│ │ 原因：检验轮次达上限 · 资料已留档   │ │
│ │ [ 回看记录 ]                      │ │

├─────────────────────────────────────┤
│ 首页              历史              我的 │
└─────────────────────────────────────┘
```

关键交互：

- 完整历史列表只在此页展示；工作台不嵌完整历史，避免干扰当前问诊。
- 列表按时间分组，卡片突出状态、主诉、当前阻塞点或终止原因。
- “继续就诊”恢复原会话和原流程状态。
- “发起复诊”读取该次完整诊断记录作为复诊上下文。
- “回看记录”进入只读详情；只读状态不展示可发送输入框，不触发 Agent 主循环。
- 搜索和筛选面向患者可理解信息：症状、诊断、状态、日期，不要求患者知道会话 ID。

空状态：

- 首次使用（无任何就诊记录）时，列表区域展示空状态插画和引导文案："还没有就诊记录，开始你的第一次问诊吧"，附带一个"新建问诊"按钮直接跳转 `/workbench/new`。
- 筛选无结果时展示"没有找到匹配的记录"，不展示新建问诊入口（因为患者是在找旧记录，不是想建新会话）。

## 3. 工作台系列

### 3.1 工作台通用框架

```text
┌─────────────────────────────────────┐
│  医  AI导诊医生         [暂离]  X     │
│     在线 · 新出诊                    │
├─────────────────────────────────────┤
│ 本次上下文  已读取病史  [展开]          │
├─────────────────────────────────────┤
│                                     │
│  AI/患者消息、流程卡片、系统事件        │
│                                     │
├─────────────────────────────────────┤
│ 快捷辅助：随阶段变化，可自动隐藏         │
├─────────────────────────────────────┤
│ [ + ] [ 输入框................ ] [发送] │
└─────────────────────────────────────┘
```

顶栏布局优先级（375pt 移动端）：

顶栏可用宽度约 345pt（去除左右安全边距）。各元素优先级互斥显示规则：

| 优先级 | 元素 | 宽度预算 | 显示条件 |
| --- | --- | --- | --- |
| 固定 | AI头像+名称 | ~120pt | 始终显示，名称过长截断 |
| 固定 | 退出 X | ~32pt | 始终显示 |
| P1 | 超时文案 | ~140pt | 剩余 ≤5min 时出现，挤占暂离按钮位置 |
| P2 | 暂离按钮 | ~48pt | 非超时警示期且非终止态时显示 |
| P3 | 急症守护图标 | ~24pt | 始终显示，仅图标无文字 |

关键约束：
- "急症守护"在移动端只显示盾牌图标（24pt），不显示文字。首次 Tooltip 悬浮在图标下方。
- 超时文案出现时，暂离按钮隐藏（此时暂离无意义，因为即将超时）。
- 副标题行（"在线 · 新出诊"）在极窄屏幕（<360pt）可隐藏，状态信息由上下文摘要栏承载。
- PC 端（≥768pt）无需互斥，所有元素均可并排显示。

倒计时规则：

- 默认不显示倒计时数字，避免增加患者焦虑。
- 剩余时间 ≤ 5 分钟时，顶栏右侧以低对比度、小字号、模糊化样式温和展示"问诊时间即将结束"文案，不使用倒计时秒数。
- 剩余时间 ≤ 2 分钟时，文案变为"即将超时，请尽快完成"，略微提高对比度但仍不使用醒目颜色。
- 超时后由系统触发 `WB-TIMEOUT` Overlay，不在计时器层面制造压迫。

急症守护标识规则：

- "急症守护"图标/文字对首次使用的患者可能不易理解。首次进入工作台时，以 Tooltip 短暂浮现说明："系统会持续监测危急信号，如发现紧急情况将立即打断并提示就医"。
- Tooltip 只展示一次（按设备/账号记录），后续不再弹出。
- 图标本身不可点击触发打断（打断由系统自动判定），但点击可再次查看说明文字。

暂离规则：

- 顶栏提供"暂离"按钮，点击后暂停总计时，进入暂离状态。
- 暂离无时间上限，患者可按需暂停任意时长；暂离期间显示温和提示"已暂停，等你回来"。
- 患者主动点击"继续"恢复计时；不设自动恢复，避免患者不在场时无声消耗时间。
- 暂离期间急症守护仍然生效（基于已有信息的后台分析不暂停）。

移动端上下文规则：

- 默认只显示一行“本次上下文摘要”，例如“新出诊 · 已读取病史”或“复诊 · 引用上次急性咽炎记录”。
- 点击“展开”打开小型 Drawer/Accordion，仅展示本次相关摘要：入口类型、过敏史提示、上次诊断摘要、未完成事项。
- 不在工作台展示完整历史列表；完整历史必须回到 `HOME-HISTORY`。
- 如果患者从历史进入复诊，上下文摘要应告诉患者“已引用哪次记录”，避免患者担心 AI 没看旧资料。

输入辅助规则：

- 输入框为空、未出现阻塞卡、患者处于初始描述或 AI 追问时，显示快捷 chip。
- 患者开始输入、发送后、上滑回看、卡片阻塞、Overlay 出现时，快捷 chip 自动收起。
- chip 的作用是填入草稿、追加短语或作为快速回答；除非文案明确为按钮动作，否则不直接提交。
- “+”入口打开补充资料 Sheet，可添加体温、持续时间、疼痛程度、图片/报告等；本期可先设计入口，不强制实现附件上传。

### 3.2 `WB-LOADING` 上下文加载态

```text
┌─────────────────────────────────────┐
│  医  AI导诊医生       急症守护  --:--  X │
│     正在准备本次问诊                  │
├─────────────────────────────────────┤
│                                     │
│ ┌─────────────────────────────────┐ │
│ │ 正在读取就诊上下文                │ │
│ │ ✓ 身份核验                        │ │
│ │ ✓ 病历病史                        │ │
│ │ ○ 上次诊断记录（复诊时）           │ │
│ │ ○ 创建本次会话                    │ │
│ │                                 │ │
│ │ [ 取消，返回首页 ]                 │ │
│ └─────────────────────────────────┘ │
│                                     │
│ 若加载的是复诊：                      │
│ ┌─────────────────────────────────┐ │
│ │ 将引用：昨天急性咽炎诊断与用药记录  │ │
│ └─────────────────────────────────┘ │
├─────────────────────────────────────┤
│ 正在加载，暂不可输入                  │
├─────────────────────────────────────┤
│ [ 请稍候...................... ] [--] │
└─────────────────────────────────────┘
```

关键交互：

- 新出诊加载患者基础信息、病历病史、既往史、过敏史。
- 复诊额外加载上次完整诊断记录、检验结果、处置和随访信息。
- 加载成功进入 `WB-CHATTING`；若是从首页症状 chip 进入，加载后把该症状放入输入草稿或首屏提示。
- 加载过程中提供"取消，返回首页"按钮，患者可随时中断加载回到首页。
- 加载失败显示重试和返回首页；不单独新增界面 ID。

```text
│ │                                 │ │
│ │ [ 取消，继续就诊 ] [ 确认退出 ]     │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

退出后果动态规则：

Sheet 只展示与患者当前所处阶段匹配的那一条后果，不罗列所有可能性：

| 当前阶段 | 展示文案 |
| --- | --- |
| 未产生任何费用 | "本次问诊将直接结束，不产生费用。" |
| 已缴费但检验/治疗未执行 | "已支付的 ¥xx 将原路退回，通常 1-3 个工作日到账。" |
| 检验/治疗已执行 | "已完成的检验/治疗费用不可退，结果会留档供下次使用。" |
| 药品已取/已配送 | "已取药品按退药政策处理，详见结算明细。" |

文案使用患者能理解的具体金额和时间，避免罗列抽象规则。

结算完成态：

```text
┌─────────────────────────────────────┐
│ ┌─────────────────────────────────┐ │
│ │ 就诊已结束 / 挂起                 │ │
│ │ 已按当前阶段完成结算并保存快照。    │ │
│ │ [ 完成 ]                         │ │
│ └─────────────────────────────────┘ │
└─────────────────────────────────────┘
```

关键交互：

- 顶栏退出按钮在任意非终止状态触发 Sheet。
- 取消返回原状态，不改变时间线。
- 确认退出触发退出结算，结算依据不可逆动作。
- 结算完成后保存快照，历史可回看。
- 急症命中时，急症 Overlay 优先于退出 Sheet。

组件标注：HeroUI Modal/Drawer；TanStack Query 用于退出结算 mutation。

## 5. 输入辅助设计

输入辅助不是独立页面，但它直接影响患者能否顺利开始问诊。

| 场景 | 展示内容 | 触发 | 消失条件 |
| --- | --- | --- | --- |
| 首次进入问诊 | 常见症状 chip：发烧、咳嗽、咽痛、头痛、腹痛等 | `WB-CHATTING` 初始空输入 | 患者开始输入、发送、滚动回看、进入阻塞卡 |
| 患者不知道怎么说 | “我不知道怎么描述” chip | 初始空输入 | 点击后进入引导式追问；引导中输入框上方常驻”直接打字描述”链接可退出引导 |
| AI 问是否有某症状 | 有/没有/不确定 chip | AI 追问生成 | 回答后消失 |
| AI 问时长/程度 | 今天开始、1-3 天、超过 3 天、轻微/中等/严重 | AI 追问生成 | 回答后消失 |
| 需要客观补充 | 体温、疼痛评分、用药史、图片/报告入口 | 点击输入框左侧 `+` | Sheet 关闭后消失 |
| 完成后复诊 | 症状加重、出现新症状、药后不适 | `WB-COMPLETED` | 触发复诊后消失 |

规则：

- chip 只帮助组织语言，不直接替患者确认医疗决策。
- 与阻塞卡动作冲突的 chip 必须隐藏。
- chip 文案使用患者语言，不使用内部状态名。
- 快捷输入不能制造事实；例如不能默认填”无呼吸困难”，必须由患者点击确认。

Chip 行为分类：

系统存在两种 chip，在视觉样式上必须可区分，避免患者无法预判点击后果：

| 类型 | 视觉样式 | 点击行为 | 适用场景 |
| --- | --- | --- | --- |
| 草稿 chip | 描边/浅色底，右侧带铅笔小图标 | 填入输入框草稿，患者可编辑后手动发送 | 常见症状入口、开放式补充（如”发烧””咳嗽”） |
| 快速回答 chip | 实心/深色底，无额外图标 | 点击即发送，等效于患者输入该文案并按发送 | 封闭式追问回答（如”有””没有””今天开始””1-3天”） |

设计约束：
- 草稿 chip 和快速回答 chip 不能混合在同一行出现，避免患者在同一组 chip 中遇到不同行为。
- AI 生成快速回答 chip 时，文案必须完整可直接发送（如”没有呼吸困难”而非”没有”），确保点击后患者可在时间线看到完整回答。
- 首次使用时，快速回答 chip 组上方显示一次性提示：”点击即发送回答”，后续不再重复。

引导式追问退出：

- 点击”我不知道怎么描述”后进入引导模式（AI 逐步追问部位→时长→程度→伴随症状），引导中输入框上方始终显示一个”直接打字描述”文字链接。
- 点击该链接后退出引导模式，恢复为普通自由输入态，已回答的引导信息保留在时间线中（不丢失）。
- 退出引导不等于结束问诊，AI 会基于已收集的信息继续追问或转入分析。

```text
  ├─ 复诊类输入/chip ────────> WB-LOADING 或 WB-CHATTING(复诊)

  └─ 确认 ─────────────────> WB-TRANSFER
```

## 7. PC 端适配说明

移动端（375pt）是主设计基线，PC 端（≥768pt）在以下方面做适配：

| 区域 | 移动端 | PC 端（≥768pt） |
| --- | --- | --- |
| 工作台整体 | 全屏单列 | 居中主列（max-width 640px）+ 右侧摘要侧栏（240px，可折叠） |
| 顶栏 | 元素互斥显示 | 所有元素并排，无需互斥优先级 |
| 常见症状 chip | 最多两行 | 可展开为三行或网格，因为屏幕空间充裕 |
| 快速回答 chip | 横向滚动或折行 | 直接展开，不需要滚动 |
| 锁定栏 | 底部固定 | 卡片内联显示，底部输入区不被锁定栏占用 |
| 输入辅助区 | 在输入框上方 | 在输入框上方，宽度跟随主列 |
| 上下文摘要 | 收起为一行 | 侧栏常驻，展示本次摘要（入口类型、过敏提示、已完成步骤） |
| 阻塞卡 | 全宽 | 主列宽度内居中，卡片最大宽度 520px |
| 全局 Overlay | 全屏遮罩 + 居中弹窗 | 同移动端，弹窗最大宽度 480px |

约束：

- PC 端侧栏只展示本次会话摘要，不展示完整历史列表。完整历史仍由 `HOME-HISTORY` 承载。
- 侧栏折叠后变为浮动按钮，点击展开；默认首次访问展开，后续记住用户偏好。
- 所有交互行为（chip 点击行为、卡片动作、锁定规则）PC/移动端完全一致，只是布局位置不同。
- 首页在 PC 端同样保持极简——不因为屏幕大就填充更多内容。

## 8. 覆盖检查

- 首页覆盖新出诊、常见症状入口、继续就诊、最近一次复诊提醒和完整历史入口。
- 完整历史集中在 `HOME-HISTORY`；工作台移动端历史默认收起，仅展示本次上下文摘要。
- 工作台覆盖上下文加载、初始问诊、快捷症状 chip、快速回答、补充资料入口、Agent 分析、阻塞卡、检验缴费、结果回填、确诊、处置、完成。
- 阻塞卡覆盖同意、不查、暂不决定、查看说明、缴费成功/失败/暂缓和处理留痕。
- 全局机制覆盖急症守护、整次导诊单一超时和主动退出结算。
- 终止态覆盖急症终止、超时/转大医院升级、追问或检验上限、本院能力不足。
- 完成后继续输入和复诊 chip 覆盖自动复诊，不把新输入追加到已完成会话。
- ASCII 草图保持原型的固定顶栏、时间线混排、锁定栏、底部输入、Overlay 和 Sheet 结构，同时补充患者输入减负和渐进式信息架构。
