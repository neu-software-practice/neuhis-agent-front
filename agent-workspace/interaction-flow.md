# 无人医院·患者端流程设计

## 一、主流程

急症判断独立成线(不占主链路菱形);患者主动退出任意步骤可触发,引向统一退出处理。复诊与新出诊在入口处区分读取的资料。

```mermaid
flowchart TD
    A[打开应用 / 身份核验] --> EN{入口类型}
    EN -->|新出诊| B1[读取病历病史]
    EN -->|复诊| B2[读取病历病史<br/>+ 上次完整诊断记录]
    B1 --> C[AI 问诊 + 症状录入]
    B2 --> C
    C --> F{是否需要检验?}
    F -->|无需检验| J[查看诊断与处置方案]
    F -->|需要检验| S[[检验子流程]]
    S --> J
    J --> K{处置方案类型}
    K -->|用药| L[药品缴费] --> M[药房取药 / 配送]
    K -->|治疗| N[预约并接受自动化治疗]
    K -->|仅医嘱| O[查看医嘱与注意事项]
    M --> P[随访 / 复诊提醒]
    N --> P
    O --> P
    P --> Q{病情是否变化?}
    Q -->|康复达标| Z([流程结束])
    Q -->|新症状·点复诊| EN

    %% 急症独立成线
    G[急症守护 · 全程监听] -. 命中急症 .-> ER[提示立即前往急诊]

    %% 患者主动退出:任意步骤可触发
    C -. 主动退出 .-> X[[退出处理]]
    J -. 主动退出 .-> X
    K -. 主动退出 .-> X
    L -. 主动退出 .-> X
    P -. 主动退出 .-> X
    S -. 主动退出 .-> X
```

## 二、检验子流程

```mermaid
flowchart TD
    S0([Agent 下发检验项]) --> G[缴纳检验费]
    G --> H[检验科采样 / 执行]
    H --> I[生成检验结果]
    I --> S9([检验结果交回 Agent])

    %% 患者主动退出(唯一离开途径,非判断)
    G -. 主动退出 .-> XOUT[[退出处理]]
    H -. 主动退出 .-> XOUT
    I -. 主动退出 .-> XOUT
```

## 三、整次导诊的单一超时计数

全流程只有这一处超时,作用于"一次完整导诊"整体,而非每个步骤各自计时。

```mermaid
flowchart TD
    T0[导诊流程开始 · 启动总计时] --> RUN[问诊 / 检验 / 处置 正常推进]
    RUN --> CHK{是否超总时限?}
    CHK -->|未超时| RUN
    CHK -->|已超时| TIP[提示前往大医院就诊]
    RUN --> DONE([流程正常结束 · 停止计时])
```

## 四、处置决策


```mermaid
flowchart TD
    A[已确诊] --> B{处置方式}
    B -->|用药| E[开具药品 + 用药提醒]
    B -->|治疗| GG{本院是否具备能力?}
    GG -->|具备| H[安排自动化治疗]
    GG -->|不具备| BIG[提示前往大医院就诊]
    B -->|仅调养| J[给出医嘱]
```

## 五、急症守护 Agent

独立并行,全程监听患者每一次新信息(每轮对话/体征/检验结果),命中即刻打断任何流程。命中后动作简化为单一提示前往急诊,不含系统侧急救/转运编排。

```mermaid
flowchart TD
    SRC[监听患者任意环节的新信息] --> SCAN[急症红旗筛查]
    SCAN --> HIT{是否命中急症?}
    HIT -->|否| SRC
    HIT -->|是| BREAK[立即打断当前流程]
    BREAK --> ALERT[提示患者:情况紧急<br/>请立即前往急诊]
```

与主流程的关系(罩在其上,而非串在其中):

```mermaid
flowchart LR
    subgraph AGENT[急症守护 Agent · 全程并行]
        W[随时筛查 · 命中即打断]
    end
    subgraph MAIN[主就诊流程]
        direction TB
        M1[问诊] --> M2[检验] --> M3[处置] --> M4[随访]
    end
    M1 -.每次输入.-> W
    M2 -.每份结果.-> W
    M4 -.状态变化.-> W
    W ==命中急症 立即打断==> ER([提示前往急诊])
```

> 急症判断**每次新信息都重新筛查**,而非开头判一次——患者可能在多轮问诊、检验后甚至随访中才暴露危急信号。

## 六、患者主动退出 · 退出处理

```mermaid
flowchart TD
    X0([患者主动退出]) --> CF{确认是否结束本次就诊?}
    CF -->|取消,继续| BACK([返回原步骤])
    CF -->|确认退出| STG{当前已发生哪些不可逆动作?}
    STG -->|未缴费/未执行| F1[直接结束 · 无需结算]
    STG -->|已缴费未执行| F2[原路退款]
    STG -->|已执行检验/治疗| F3[费用照收 · 结果留档]
    STG -->|已取药/已配送| F4[按退药政策处理]
    F1 --> SAVE[保存就诊快照]
    F2 --> SAVE
    F3 --> SAVE
    F4 --> SAVE
    SAVE --> END([就诊结束/挂起])
```

> 退出后果由"已发生的不可逆动作"决定,而非由停在第几步决定。

## 二、Agent 决策主循环

主流程里的「问诊→是否需要检验→确诊」本质是同一套推理在反复运转:每次汇总最新信息(含新追问的回答、新返回的检验结果)判断是否足够确诊,不够则补、够则确诊转处置。追问与检验各设轮次上限,达上限统一提示前往大医院。急症由独立守护并行处理,不在本循环内。

```mermaid
flowchart TD
    IN[汇总现有信息<br/>病史 + 对话 + 已有检验结果] --> ANA[分析病情<br/>梳理可能病症范围]
    ANA --> SUF{信息是否足够确诊?}

    SUF -->|缺症状细节| AL{追问是否达轮次上限?}
    AL -->|未达| ASK[向患者追问<br/>优先最能区分病症的问题]
    ASK --> IN
    AL -->|已达| BIG[提示前往大医院就诊]

    SUF -->|缺客观依据| TL{检验是否达轮次上限?}
    TL -->|未达| TEST[下发检验项<br/>明确鉴别目标]
    TEST --> EXE[[检验子流程 · 纯执行]]
    EXE --> RES[读取检验结果]
    RES --> IN
    TL -->|已达| BIG

    SUF -->|足够确诊| DX[给出诊断]
    DX --> PLAN[进入处置决策]
```
